# ⚠️ v2.0-beta - 实验阶段脚本

**警告**：此版本尚未在真实小红书页面上验证。不推荐用于研究。

## 已知问题

1. **note_id 正则表达式错误**
   - 预期：接受含字母的笔记 ID，如 `69a016780000000022039019`
   - 当前：仅接受 15-18 位纯数字
   - 结果：真实笔记 ID 会被识别为 `null`，导致 `note_url` 也为 null

2. **正文选择器不完整**
   - 缺少 `#detail-desc`、`.desc` 等实测有效的选择器
   - 可能导致 `content` 字段为 null

3. **互动量解析失败**
   - 当前算法要求"数字＋赞/收藏等关键词"的格式
   - 实际主互动栏只显示图标和数字，无文字标签
   - 可能导致 `likes`、`collects` 等全部为 null

4. **登录检测误判**
   - 统计隐藏的 `[class*="login"]` 元素
   - 正常页面可能包含隐藏元素，导致假阳性

5. **预览 XSS 风险**
   - 使用 `innerHTML` 拼接页面内容
   - 应改为安全的 `textContent`

6. **文档不一致**
   - README 建议验证码后重试
   - 但规范要求"出现验证码即停止，不重试"

## 如何改进

若想继续开发此版本，请：

1. 用真实笔记 `69a016780000000022039019` 测试
2. 修复上述 6 个问题
3. 验证输出为 `likes=831, collects=479, comments=256, shares=null`
4. 通过安全审查后再标记为 stable
